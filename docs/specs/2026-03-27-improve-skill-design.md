# Improve Skill — Design Spec

**Дата:** 2026-03-27
**Статус:** Согласован

## Назначение

Интерактивное улучшение содержимого заметки Obsidian — структура, форматирование, текст, связи. Пользователь выбирает операции из меню с оценкой релевантности, скилл предупреждает о конфликтах порядка и выполняет по одной с превью и подтверждением.

Применяется к любой заметке: как из `_inbox/`, так и из `notes/`, `literature/`, `projects/`.

## Операции

Шесть операций, упорядоченных по безопасному приоритету (от структуры к содержанию):

| # | Операция | Что делает |
|---|----------|-----------|
| 1 | **Structure** | Добавление/реорганизация заголовков. Пустые секции: спрашивает — удалить, оставить или заполнить из контекста |
| 2 | **Format** | Выделение терминов (bold), списки, цитаты, callouts — единообразное форматирование |
| 3 | **Tabulate** | Превращение перечислений и сравнений в markdown-таблицы |
| 4 | **Rewrite** | Улучшение формулировок, убирание воды, повышение ясности |
| 5 | **Summarize** | Сжатие до ключевых тезисов, добавление TL;DR |
| 6 | **Wikify** | Добавление `[[wikilinks]]` на существующие заметки в хранилище (точные совпадения + очевидные синонимы/аббревиатуры) |

## Флоу работы

### 1. Вызов

Пользователь вызывает `/improve` и указывает имя или путь к заметке.

### 2. Анализ

Скилл читает заметку и для каждой из 6 операций определяет релевантность: высокая / средняя / низкая / не нужна.

### 3. Меню

Показывает список операций с оценкой релевантности и кратким обоснованием. Пользователь выбирает номера через запятую.

Пример:
```
Improve: Стрим ТК ПРМ

[1] Structure  — высокая (3 пустые секции, неструктурированный текст)
[2] Format     — средняя
[3] Tabulate   — средняя (1 кандидат)
[4] Rewrite    — высокая (2 фрагмента)
[5] Summarize  — низкая (заметка <2000 слов)
[6] Wikify     — средняя (4 кандидата)

Выбери номера через запятую (например 1,3,6)
```

### 4. Проверка порядка

Если выбранные операции конфликтуют по порядку (например, Summarize перед Rewrite), скилл показывает предупреждение и рекомендуемый порядок. Пользователь подтверждает или меняет.

### 5. Выполнение

По одной операции:

1. Показывает превью изменений (было / стало)
2. Пользователь: применить / отменить / скорректировать
3. Если применено — изменения записываются в файл
4. Переход к следующей операции (работает с уже обновлённым текстом)

### 6. Итог

- Отчёт: что применено, что пропущено
- Предложение записать в Daily Note

## Safety-правила

- **Никогда не перезаписывать без превью** — каждое изменение показывается как «было / стало» до применения
- **Сохранять frontmatter** — скилл не трогает YAML-блок, только тело заметки
- **Сохранять wikilinks** — существующие `[[ссылки]]` не удаляются и не переименовываются
- **Сохранять TODO-чекбоксы** — `- [ ]` элементы остаются как есть
- **Wikify только по существующим заметкам** — проверка через `obsidian search`, плюс очевидные синонимы/аббревиатуры
- **Не сжимать таблицы** — Summarize не трогает табличные данные
- **Язык заметки** — скилл определяет язык текста и работает на нём, не переводит
- **Откат** — хранит копию исходного текста, может откатить все изменения текущей сессии

## Интеграция с другими скиллами

### Фаза 1 (текущая): Standalone

Скилл `improve` работает независимо, вызывается пользователем.

### Фаза 2 (позже): Рекомендации из других скиллов

- **audit** (режим `full`) — если находит заметки с низким качеством содержимого (пустые секции, стена текста), рекомендует: `/improve <имя заметки>`
- **process-inbox** — после обработки заметки, если содержимое неструктурировано, предлагает запустить `/improve`
- **decompose** — если `improve` видит заметку >10KB или >2 тем верхнего уровня, предупреждает: рекомендуется сначала `/decompose`
